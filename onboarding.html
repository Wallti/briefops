Preparation · What you provide

What we need
from you.

BriefOps is configured around your actual organisation. The quality of the output depends directly on the quality of the input. Here is what you need to prepare — and what is optional.

Required · Day 1

Org-chart & team structure

A list of your teams, their leads, and how they depend on each other. Doesn't need to be formal — a spreadsheet or Notion table is fine.

Team names and responsible leads
Who approves what (sign-off chains)
Cross-team dependency rules (e.g. Design before Engineering)
Email addresses for delivery routing

Required · Day 1

Integration credentials

API keys and access tokens for the tools BriefOps will write to. We provide a credential checklist during onboarding.

Jira / Monday / Linear / ClickUp API token
Slack Bot token + target channels
Google Calendar or Outlook OAuth consent
SMTP credentials for email delivery (optional)

Required · Day 1

Three sample meetings

Transcripts, memos, or notes from three representative meetings. These calibrate the parser to your team's language and terminology.

At least one sprint planning or project meeting
At least one cross-team meeting
Any format: transcript, audio file, or written notes
Anonymised if needed — content only

Optional · Improves output quality

Existing sprint / task history

A Jira or Monday export of the last 2–3 sprints. Used to calibrate story point estimates and identify team capacity patterns.

CSV export from Jira / Monday / Linear
Last 2–3 sprints sufficient
Used only for estimation calibration

Optional · Advanced tier only

Document library

Key reference documents — policies, previous decisions, product specs — that the Knowledge Base agent should be aware of.

PDF or Word format
Privacy level assigned per document (full / reference-only)
Maximum 500MB for initial ingestion

Optional · Standard + Advanced

Approval workflow rules

Who needs to approve which types of tasks before they go to Jira. Default is team lead approval for all tickets — custom rules can be configured.

Approval thresholds by task type or story points
Escalation rules for blocked tasks
Auto-approve rules for routine task types

Implementation · Day by day

What the first week
looks like.

This is the Starter timeline (7 days). Standard adds dependency setup and multi-channel delivery in days 5–10. Advanced adds knowledge base, executive digest, and capacity analysis in days 11–25.

Day 01 Kickoff

Contract signed. Kickoff call. Credentials collected.

90-minute kickoff call. We walk through your org-chart together, configure the initial team mapping, and verify all integration credentials. By end of day, the org-chart JSON is configured and credential checks pass.

BriefOpsSend credential checklist + org-chart template
ClientComplete org-chart template, provide API credentials
BriefOpsConfigure team routing and verify integration connections
Both sides · ~3h total
Day 02 Calibration

Pipeline calibration on your sample meetings.

We run your three sample meetings through the full pipeline. You review the extracted tasks and flag any routing errors or terminology mismatches. The parser is adjusted to your team's language.

BriefOpsRun sample meetings through parser, deliver output report
ClientReview output, flag errors (1h review time)
BriefOpsAdjust parser configuration based on feedback
~2h client time
Day 03 Staging

Staging environment. End-to-end test.

Full pipeline test in staging: meeting input → task extraction → Jira staging board → Slack staging channel → calendar test event. You verify the output looks correct in each target system.

BriefOpsRun end-to-end test on staging
ClientVerify Jira staging board, Slack messages, calendar entries (~30min)
BriefOpsFix any formatting or routing issues
Mostly us · 30min client
Day 04–05 Team training

Team onboarding. First live meeting.

45-minute team walkthrough: how to submit a meeting, how to review and approve tasks, how to use the web interface. The first real meeting is processed live during the session. Team leads receive the approval workflow guide.

BriefOpsRun team training session (45min video call)
ClientSubmit first real meeting during session
ClientTeam lead approves first ticket batch
45min all-team session
Day 07 Go-live

Production. Full handover.

System moves to production. You receive full documentation, the admin panel, and direct contact for the first 30 days. A 30-day check-in call is scheduled automatically.

BriefOpsProduction deployment, handover documentation
BriefOpsAdmin access, monitoring dashboard access
Both30-day check-in call scheduled
Handover complete

Standard + Advanced · Additional steps

What changes
for higher tiers.

Standard · Days 5–14

→Dependency graph configuration — cross-team blocking rules set up based on your org-chart
→Multi-channel delivery tuning — Slack channels mapped per team, email templates reviewed
→Approval workflow configuration — who approves what, escalation thresholds
→Blocker detection calibration — test cases for your most common cross-team dependencies

Advanced · Days 11–25

→Knowledge base initial ingestion — document library processed and indexed
→Document privacy classification — each document assigned full / reference-only / invisible
→Capacity analysis calibration — sprint history loaded, estimation model trained on your data
→Executive digest configuration — recipients, format, delivery schedule
→Follow-up agent calibration — escalation thresholds tuned to your team's working rhythm
