Self-Hosted Setup

Up and running
in four steps.

The BriefOps Docker stack includes the Python backend, FastAPI server, LightRAG knowledge base, Ollama for local LLMs, and the web frontend. Your team accesses it through any browser on your internal network.

01

Clone and configure

BriefOps ships with a preconfigured docker-compose.yml. Copy the environment template and fill in your API keys and integration credentials.

# Clone the repository git clone https://github.com/yourorg/briefops.git cd briefops # Copy environment template cp .env.example .env # Fill in your credentials nano .env
02

Configure your org-chart

BriefOps needs to know your team structure to route tasks correctly. Edit the org-chart JSON with your actual teams, owners, and dependency rules. This is the only configuration file you need to touch.

# Edit your team structure nano config/org_chart.json # Example structure { "teams": [ {"id": "engineering", "lead": "Tom H."}, {"id": "design", "lead": "Sara L."} ] }
03

Start the stack

One command starts all services: the API server, knowledge base, local LLM (if configured), frontend, and monitoring dashboard.

# Start all services docker compose up -d # Check status docker compose ps # View logs docker compose logs -f briefops-api
04

Connect your integrations

Open the web interface on your internal network and connect Jira, Slack, and your calendar. The system validates each connection and runs a test meeting through the full pipeline before go-live.

# Access the web interface http://your-server:8080 # API documentation http://your-server:8000/docs # Health check curl http://your-server:8000/health

DSGVO · Compliance by industry

What your industry
actually requires.

The DSGVO applies to everyone. But some industries have additional requirements that interact directly with how BriefOps processes meeting content.

Agencies · Product teams · Consultancies

Standard compliance — simplest path

These teams can run BriefOps on managed cloud with a standard DPA and SCCs. No additional regulatory requirements beyond baseline DSGVO.

Standard DPA + SCCs sufficient
AWS Frankfurt (EU) satisfies data residency
No sector-specific regulation
Finance · Insurance · Asset management

MaRisk + BaFin outsourcing rules

AI processing qualifies as outsourcing under MaRisk and BaFin guidelines. Requires formal outsourcing agreement, exit strategy documentation, and audit rights. DORA applies from 2025 for ICT risk management.

Outsourcing agreement required (BaFin)
Exit strategy + audit rights documented
DORA ICT risk assessment from 2025
Self-Hosted Docker resolves most requirements
Legal · Tax advisory · Notaries

§203 StGB — professional secrecy

Client-related meeting content falls under §203 StGB professional secrecy obligations. Technically, no client-identifying data may be processed in a cloud system. Fully local deployment is required.

§203 StGB — no client data in cloud
Anonymisation before cloud processing not sufficient
Fully local deployment required
Air-gapped option available
Healthcare · Hospitals · Medical practices

Art. 9 DSGVO — special categories

Patient data falls under Art. 9 DSGVO (special categories) and national health data laws. Meeting content with patient references must be excluded from AI processing entirely, or a fully local setup must be used.

Art. 9 DSGVO — explicit legal basis required
Patient data cannot be processed in cloud AI
Technical exclusion of patient references required
Fully local + Reference-only privacy mode
Public sector · Municipalities · Agencies

BSI IT-Grundschutz + C5

Public sector procurement requires BSI IT-Grundschutz compliance. AWS Frankfurt holds BSI C5 certification. Self-Hosted Docker on BSI-certified infrastructure satisfies procurement requirements.

BSI IT-Grundschutz compliance required
Art. 35 DSGVO — DSFA mandatory
AWS Frankfurt BSI C5 certified
Self-Hosted on Bundescloud possible
Technology · SaaS · E-Commerce

Baseline DSGVO — straightforward

Tech companies typically have mature data handling processes and no sector-specific obligations beyond standard DSGVO. Cloud deployment with DPA is sufficient for most cases.

Standard DPA sufficient
No sector-specific regulation
Cloud or self-hosted — free choice

This overview is for general orientation. BriefOps is not a law firm. Please consult your data protection officer or legal counsel before go-live in regulated environments.
