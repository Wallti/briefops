Self-hosted setup

Up and running
in four steps.

The Docker stack includes the Python backend, FastAPI server, LightRAG knowledge base, Ollama for local LLMs, and the web frontend. Your team accesses it through any browser on your internal network.

01
Clone and configure
BriefOps ships with a preconfigured docker-compose.yml. Copy the environment template and fill in your API keys.
# Clone the repository git clone https://github.com/yourorg/briefops.git cd briefops # Copy environment template cp .env.example .env nano .env
02
Configure your org-chart
Edit the org-chart JSON with your actual teams, owners, and dependency rules. This is the only configuration file you need to touch.
{ "teams": [ {"id": "engineering", "lead": "Tom H."}, {"id": "design", "lead": "Sara L."} ] }
03
Start the stack
One command starts all services — API server, knowledge base, local LLM, frontend, and monitoring dashboard.
# Start all services docker compose up -d # Health check curl http://your-server:8000/health
04
Connect your integrations
Open the web interface on your internal network and connect Jira, Slack, and your calendar. The system runs a test meeting through the full pipeline before go-live.
# Access the web interface http://your-server:8080 # API documentation http://your-server:8000/docs
